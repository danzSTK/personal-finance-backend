# Suporte

Use a tabela abaixo para escolher o canal correto. Isso ajuda a manter informações públicas organizadas e relatos sensíveis protegidos.

| Necessidade                   | Canal                                                                                                                                               |
| ----------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------- |
| Bug reproduzível              | [Template público de bug](https://github.com/danzSTK/personal-finance-backend/issues/new?template=bug.md)                                           |
| Sugestão de funcionalidade    | [Template de feature](https://github.com/danzSTK/personal-finance-backend/issues/new?template=feature.md)                                           |
| Incidente operacional         | [Template de incidente](https://github.com/danzSTK/personal-finance-backend/issues/new?template=incident.md)                                        |
| Vulnerabilidade               | [Private Vulnerability Reporting](https://github.com/danzSTK/personal-finance-backend/security/advisories/new)                                      |
| Dúvida sobre execução local   | [Documentação](./docs/getting-started.md) ou [issue de suporte](https://github.com/danzSTK/personal-finance-backend/issues/new?template=support.md) |
| Parceria ou licença comercial | [E-mail de contato](mailto:danielfelix.dev@gmail.com)                                                                                               |
| Contribuição de código        | Solicite autorização prévia pelo [e-mail do autor](mailto:danielfelix.dev@gmail.com)                                                                |

## Antes de solicitar suporte

- consulte a [documentação técnica](./docs/README.md);
- procure por issues existentes sobre o mesmo assunto;
- descreva o ambiente, o comportamento observado e o resultado esperado;
- remova secrets, tokens, credenciais, dados financeiros e informações pessoais.

Vulnerabilidades nunca devem ser enviadas por uma issue pública. Use exclusivamente o canal privado indicado na tabela e consulte [SECURITY.md](./SECURITY.md).

Contribuições externas de código ainda não estão abertas. Consulte [CONTRIBUTING.md](./CONTRIBUTING.md) antes de preparar qualquer modificação.
