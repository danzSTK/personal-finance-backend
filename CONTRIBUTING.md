# Participando do Danfy

Issues, sugestões, relatos de bugs e relatos responsáveis de vulnerabilidades são bem-vindos. Use sempre o canal adequado para cada tipo de contribuição.

## Sugestões e bugs

Sugestões de produto, propostas técnicas e bugs sem impacto de segurança podem ser enviados publicamente pelos [templates de issues](https://github.com/danzSTK/personal-finance-backend/issues/new/choose).

Antes de publicar:

- verifique se já existe uma issue sobre o assunto;
- descreva o contexto e o comportamento observado;
- inclua passos de reprodução quando aplicável;
- remova credenciais, tokens, dados financeiros e informações pessoais;
- não publique detalhes de uma possível vulnerabilidade.

## Vulnerabilidades

Suspeitas de vulnerabilidade devem ser enviadas exclusivamente pelo [GitHub Private Vulnerability Reporting](https://github.com/danzSTK/personal-finance-backend/security/advisories/new).

Não use issues, discussions, pull requests, redes sociais ou outros canais públicos. Consulte a [Política de Segurança](./SECURITY.md) antes de enviar o relato.

## Contribuições externas de código

Contribuições externas de código ainda não estão abertas. Este documento não concede permissão para modificar, distribuir ou criar trabalhos derivados do código da Danfy.

Não abra pull requests com alterações de código, documentação, configuração ou assets sem autorização prévia e expressa do autor. A disponibilidade pública do código e dos templates do GitHub não constitui autorização para contribuir com modificações.

Se deseja ajudar no desenvolvimento, entre em contato com **Daniel Félix** pelo e-mail [danielfelix.dev@gmail.com](mailto:danielfelix.dev@gmail.com). Uma colaboração autorizada poderá exigir termos adicionais antes do início do trabalho.
